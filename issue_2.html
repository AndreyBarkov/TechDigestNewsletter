The Power of Ten – Rules for Developing Safety-Critical Code 

 

 

 

 

You probably heard about the new rover Perseverance making a successful landing on Mars. When the rover was entering Mars's atmosphere, it had to go through "7 minutes of terror" when events are happening faster than the radio signal can reach the command center on Earth to receive any instructions, which means the landing has to be fully automated. One tiny bug in code could lead to the failure of this 8-year long mission.

 

NASA software engineers have to write very reliable code that they have no chance to test until something is being shot out into space, so they have a specific set of rules for writing code. This document is a set of 10 rules that they follow to write reliable code.

 


 

 

 

 

How to Make Your Code Reviewer Fall in Love with You

We know all the pain that reviewing someone's code can cause. Sometimes we are the cause of pain for our peers. To break a circle of bad code review experience, we can start with ourselves, step up and apply some of the techniques proposed in this article. Let's be kind to each other.

What we can learn from software history

by Hillel Wayne

 

 

 

 

Hillel takes us on an archeology trip and shows how one can uncover how the world of Software development became as it is. In this particular example, Hillel investigates where the "Reverse a Linked List" interview question comes from and why it is still being asked today.

 


 

 

 

Mastering Chaos - A Netflix Guide to Microservices

by Josh Evans

 

 

 

 

In this talk, Josh covers how Netflix has evolved from a monolithic application for renting physical copies of the movies to a large-scale microservice application that serves millions of people today. You'll find interesting insights on building resilient and testable microservice infrastructures.

 


 

 

 

Light Years Ahead | The 1969 Apollo Guidance Computer

by Robert Wills 

 

 

 

 

 

I don't have a particular passion for vintage technology and never really dabbled in wielding hardware or writing low-level code, but talks like this spark my interest every time. You will learn how MIT engineers designed the Apollo Guidance Computer and how it worked through the prism of the Apollo 11 mission that landed on the moon over 50 years ago.
